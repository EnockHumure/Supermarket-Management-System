## 📋 FIELD MAPPING

### From Simba JSON → To Your Database

```
Simba JSON                    Your Database
─────────────────────────────────────────────────────────
{
  "id": 13001              →  barcode: "SIM013001" (generated)
  "name": "Product Name"   →  name: "Product Name"
  "price": 83600.0         →  price: 83600.00
  "category": "Food"       →  category_id: 1 (lookup)
  "subcategoryId": 13      →  ❌ IGNORED (not in your system)
  "inStock": true          →  ❌ IGNORED (you use stockQuantity)
  "image": "url..."        →  ❌ IGNORED (not implemented)
  "unit": "Pcs"            →  ❌ IGNORED (not implemented)
}
                           →  stock_quantity: 45 (random 10-100)
                           →  reorder_level: 9 (20% of stock)
                           →  product_id: auto-generated by DB
```

---

## 🎯 WHAT HAPPENS TO EACH FIELD

### ✅ **Fields We USE:**

1. **id** (from JSON)
   - Used to generate barcode
   - Format: "SIM" + 6-digit number
   - Example: id=13001 → barcode="SIM013001"

2. **name** (from JSON)
   - Copied directly to your product name
   - Special characters escaped (e.g., ' becomes '')
   - Example: "Farmer's Choice" → "Farmer''s Choice"

3. **price** (from JSON)
   - Converted from Double to BigDecimal
   - Keeps 2 decimal places
   - Example: 83600.0 → 83600.00

4. **category** (from JSON)
   - Mapped to category_id in your database
   - Creates category if doesn't exist
   - Example: "Food Products" → category_id=1

---

### 🔧 **Fields We GENERATE:**

5. **barcode** (generated)
   - Format: "SIM" + 6-digit product ID
   - Unique for each product
   - Example: SIM013001, SIM235001

6. **stock_quantity** (generated)
   - Random number between 10 and 100
   - Simulates realistic inventory
   - Example: 45, 78, 23

7. **reorder_level** (generated)
   - Set to 20% of stock_quantity
   - Triggers low stock alerts
   - Example: stock=50 → reorder=10

8. **product_id** (auto-generated by database)
   - PostgreSQL auto-increment
   - Unique identifier
   - Example: 1, 2, 3, ...

---

### ❌ **Fields We IGNORE:**

9. **subcategoryId** - Not implemented in your system
10. **inStock** - You use stockQuantity instead
11. **image** - No image storage in your system
12. **unit** - Not implemented yet

## 📊 EXAMPLE TRANSFORMATION

### Input (Simba JSON):
```json
{
  "id": 235001,
  "name": "Amstel 100% Pure Malt 330ml",
  "price": 1300.0,
  "category": "Alcoholic Drinks",
  "subcategoryId": 235,
  "inStock": true,
  "image": "https://res.cloudinary.com/.../product_235001.jpg",
  "unit": "Pcs"
}
```

### Output (SQL INSERT):
```sql
INSERT INTO product (name, barcode, price, stock_quantity, reorder_level, category_id) 
VALUES (
  'Amstel 100% Pure Malt 330ml',  -- name (from JSON)
  'SIM235001',                      -- barcode (generated)
  1300.00,                          -- price (from JSON)
  67,                               -- stock_quantity (random)
  13,                               -- reorder_level (20% of 67)
  6                                 -- category_id (Alcoholic Drinks)
);
```

### Result in Database:
```
product_id: 1 (auto-generated)
name: Amstel 100% Pure Malt 330ml
barcode: SIM235001
price: 1300.00
stock_quantity: 67
reorder_level: 13
category_id: 6
```

## 💡 WHY WE DO IT THIS WAY

### **Why generate barcodes?**
- Simba data doesn't have barcodes
- Your system requires unique barcodes
- "SIM" prefix identifies them as Simba products
- Easy to scan and search

### **Why random stock quantities?**
- Simba only has true/false for inStock
- Your system needs actual numbers
- Random values (10-100) simulate real inventory
- You can adjust them later in your app

### **Why ignore images?**
- Your Product model doesn't have image field
- Would require database schema changes
- Can be added later if needed

### **Why ignore subcategories?**
- Your system only has categories, not subcategories
- Keeps it simple for now
- Can be enhanced later
